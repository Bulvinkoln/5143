•	Пример отправления локальной ветки в удаленный репозиторий
Теперь давайте перенесем нашу локальную ветку в удаленный репозиторий. К примеру, чтобы наш коллега мог убедиться, что мы весь день работали.
git push origin "branch-name"
Аргумент origin говорит нам, что мы хотим отправить ветку "branch-name" из локального репозитория в удаленный.
Merge request / pull request — дословно слияние.
Pull request — запрос на слияние кодовой базы из одной ветки в другую.
Напрямую изменения в master не вносятся, это строго запрещено. Перед тем, как код попадет в общую кодовую базу, он должен пройти так называемое code review и тестирование.
Code review — проверка качества кода нашими коллегами. В PR можно оставлять комментарии, вести дискуссии. Когда изменения одобрены коллегами, а код протестирован, то изменения можно вливать.
Таким образом, ветвление и последующее слияние позволяет работать параллельно и одновременно нескольким участникам команды, а затем соединить всю работу обратно в единый проект, протестировав ее перед финальной презентацией
